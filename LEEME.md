# Portafolio de Competencias — Lucas Elizalde

Estructura lista para publicar en GitHub Pages.

## Cómo usar

1. Subir todo el contenido de esta carpeta a un repositorio público de GitHub.
2. Activar GitHub Pages desde `Settings > Pages > Deploy from a branch > main / root`.
3. Abrir la URL publicada.
4. En Google Sites, insertar las páginas con `Insertar > Incorporar > URL`.

## Páginas principales

- `/index.html` → Inicio
- `/sobre-mi/` → Sobre mí
- `/competencias/` → Índice de competencias
- `/competencias/operacion/` → Operación
- `/competencias/mantenimiento/` → Mantenimiento
- `/competencias/diseno/` → Diseño
- `/competencias/innovacion/` → Innovación e Investigación
- `/contacto/` → Contacto

## Evidencias

Hay 28 páginas de evidencia en `/evidencias/`.
Cada evidencia tiene esta estructura:

```text
evidencias/nombre-evidencia/
├─ index.html
├─ img/
│  ├─ imagen-1.jpg
│  ├─ imagen-2.jpg
│  └─ imagen-3.jpg
├─ videos/
│  └─ demo.mp4
└─ docs/
   └─ informe.pdf
```

Para completar cada evidencia, subí tus archivos con esos nombres exactos. Si no subís algo, el HTML muestra un espacio/placeholder o queda el enlace preparado.

## Importante sobre el nombre del repositorio

Los enlaces internos están hechos con rutas relativas, por lo que no importa cómo se llame el repositorio.

## Archivos no incluidos

No incluí PDFs, fotos ni videos reales para que el ZIP no sea pesado y para que puedas elegir los materiales definitivos.
