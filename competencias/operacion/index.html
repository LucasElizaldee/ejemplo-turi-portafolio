Operación de Sistemas Mecatrónicos

Portafolio de Competencias — Lucas Elizalde

Área de dominio

Esta área de dominio se relaciona con la capacidad de implementar, operar, monitorear y poner en servicio sistemas mecatrónicos en contextos industriales y de laboratorio. A lo largo de mi formación desarrollé esta competencia a través de prácticas donde fue necesario integrar hardware y software, verificar el funcionamiento de los sistemas, realizar mediciones y trabajar con criterios de operación y seguridad.

Índice de competencias

  1. Implementar software y hardware específico para el correcto desempeño de procesos.
  2. Verificar la capacidad operacional de sistemas mecatrónicos mediante sistemas de monitoreo.
  3. Instalar y poner en servicio sistemas mecatrónicos considerando normas de calidad, seguridad y medio ambiente.

1. Implementar software y hardware específico para el correcto desempeño de procesos

Esta competencia implica conectar, configurar y programar componentes de hardware y software para que un sistema mecatrónico cumpla una función específica. En mi caso, se evidencia en prácticas donde integré microcontroladores, sensores, actuadores, comunicación digital, interfaces de usuario y sistemas físicos que debían responder ante entradas o condiciones determinadas.

Evidencias

Proyecto Integrador II: Robot humanoide Turi

Se diseñó un robot humanoide inspirado en NAO para apoyar recorridos turísticos dentro de la universidad. El sistema integró ESP32, Arduino UNO, servomotores, ESP32-CAM, audio, aplicación móvil, servidor en Python y comunicación WiFi.

Esta evidencia muestra la implementación de hardware y software en un sistema mecatrónico completo, donde diferentes subsistemas debían operar de forma coordinada.

Proyecto Integrador I: Aromatizador automático

Se diseñó e implementó un aromatizador automático utilizando Arduino, circuito electrónico, display, botones, LEDs, buzzer y un sistema de accionamiento.

Esta actividad permitió aplicar programación embebida y control de actuadores en un prototipo funcional, desarrollado para resolver una necesidad concreta.

Ver evidencia

Comunicación SPI maestro–esclavo con ATmega328P

En esta práctica se implementó una arquitectura maestro–esclavo utilizando comunicación SPI entre dos microcontroladores ATmega328P. El maestro leía sensores y enviaba tramas de control al esclavo, que accionaba un LED PWM, un servomotor y un buzzer.

Esta actividad permitió integrar lectura de sensores, comunicación entre dispositivos y control de actuadores en un sistema embebido.

Ver evidencia

Comunicación I2C maestro–esclavo con ATmega328P

Se replicó la arquitectura de control usando I2C entre dos ATmega328P, manteniendo el sensado y accionamiento del sistema, pero utilizando un bus de comunicación diferente.

Esta práctica permitió implementar hardware y software específico, además de comprender cómo cambia la conexión y operación del sistema según el protocolo utilizado.

Ver evidencia

Control de animaciones en matriz RGB mediante UART

Se programó una matriz LED RGB 8x8 basada en WS2812B, permitiendo seleccionar animaciones mediante comandos enviados por UART. Las animaciones se organizaron en frames y fueron ejecutadas desde el microcontrolador.

Esta práctica evidencia la integración entre comunicación serial, visualización digital y programación de sistemas embebidos.

Ver evidencia

Control de movimiento en matriz RGB con MPU6050

Se implementó un sistema donde un LED se desplazaba dentro de una matriz RGB según la inclinación medida por el sensor MPU6050. También se incorporó un botón para modificar el color del LED.

Esta práctica permitió combinar sensado inercial, control en tiempo real y retroalimentación visual.

Ver evidencia

Robot de mini fútbol con Bluetooth

Se desarrolló un robot móvil de mini fútbol con tracción diferencial, sensores de línea, mecanismo frontal de golpeo y comunicación Bluetooth para control remoto.

Esta actividad integra movilidad, accionamiento, comunicación inalámbrica y operación de un sistema físico con interacción directa.

Cerradura RFID con RC522, LCD I2C y EEPROM

Se implementó una cerradura electrónica con lector RFID RC522, pantalla LCD I2C, LEDs de estado, botones de actualización/borrado y almacenamiento persistente en EEPROM.

El sistema permitía registrar, validar, modificar y conservar una tarjeta autorizada, integrando control de acceso, memoria y comunicación con el usuario.

Selector de color con fotocelda, servomotor y WS2812

Se desarrolló un sistema capaz de detectar colores mediante una fotocelda, orientar un servomotor SG90 hacia el color identificado y representar visualmente el resultado en una tira LED WS2812.

Esta práctica permitió integrar lectura analógica, calibración, control de posición y salida visual.

Piano electrónico con doble buzzer

Se implementó un piano electrónico con pulsadores asociados a notas musicales, doble buzzer y selección de canciones mediante comunicación UART.

¿Cómo desarrollé y cómo aplico esta competencia en mi vida académica?

Esta competencia la desarrollé al realizar prácticas donde el software y el hardware debían funcionar de forma conjunta. En varias actividades fue necesario leer sensores, procesar datos, generar señales de control y accionar componentes físicos.

Estas experiencias me ayudaron a comprender que operar un sistema mecatrónico no consiste solo en programar o conectar componentes, sino en lograr que todas las partes trabajen de manera coordinada para resolver una necesidad técnica concreta.

2. Verificar la capacidad operacional de sistemas mecatrónicos mediante sistemas de monitoreo

Esta competencia se relaciona con comprobar que un sistema funciona correctamente mediante mediciones, observación de señales, monitoreo de variables, análisis de datos y comparación entre el comportamiento esperado y el comportamiento real.

Evidencias

Motor trifásico con tablero de comando y medición eléctrica

Se trabajó con un motor trifásico de inducción y un tablero de comando y protección. Se verificó el funcionamiento del arranque estrella-triángulo y se realizaron mediciones de corriente y potencia eléctrica durante la operación.

La actividad permitió comprobar el desempeño del motor en condiciones reales y relacionar las mediciones con el comportamiento del sistema.

Ver evidencia

Control térmico con LM35 y monitoreo serial

Se implementó un sistema de control de temperatura con sensor LM35, calefactor y ventilador. El sistema enviaba por UART la temperatura y el estado de los actuadores.

Los datos podían ser graficados en Python, permitiendo observar la evolución térmica y la respuesta del sistema ante distintos rangos de temperatura.

Ver evidencia

Control de motor DC por referencia analógica

En esta práctica se comparó una referencia analógica con la posición actual de un motor DC, ajustando el sentido de giro y la señal PWM según el error detectado.

Mediante comunicación serial se monitoreaban valores como referencia, posición, error, PWM y dirección de giro, permitiendo verificar la respuesta operacional del sistema.

Ver evidencia

Medición de señales con instrumentos de laboratorio

Se utilizaron multímetro, fuente de alimentación, osciloscopio, generador de funciones y amperímetro para observar señales y medir variables eléctricas.

Estas actividades fueron fundamentales para verificar circuitos, detectar diferencias entre valores esperados y medidos, y validar el funcionamiento de sistemas electrónicos.

Ver evidencia

Pruebas de electrónica analógica con medición y simulación

En los laboratorios de electrónica analógica se trabajó con diodos, rectificadores, Zener, transistores, filtros y amplificadores operacionales, realizando mediciones y comparaciones con simulaciones.

Estas prácticas permitieron verificar la respuesta real de los circuitos y analizar señales mediante instrumentos de laboratorio.

Ver evidencia

¿Cómo desarrollé y cómo aplico esta competencia en mi vida académica?

Esta competencia la desarrollé al entender que un sistema no se valida solamente porque enciende o ejecuta una acción. Es necesario medir, monitorear y comparar sus variables de funcionamiento.

En mi vida académica esto me ayuda a trabajar con mayor criterio técnico, prestando atención a los datos obtenidos y a la interpretación de resultados antes de afirmar que un sistema funciona correctamente.

3. Instalar y poner en servicio sistemas mecatrónicos considerando normas de calidad, seguridad y medio ambiente

Esta competencia se vincula con la instalación, conexión, revisión y puesta en funcionamiento de sistemas de forma segura y ordenada. Incluye interpretar esquemas, respetar procedimientos, comprobar conexiones y verificar el sistema antes de considerarlo operativo.

Evidencias

Arranque estrella-triángulo de motor trifásico

Se trabajó con el arranque estrella-triángulo de un motor trifásico, verificando la actuación de contactores, temporizador, protecciones y elementos de mando.

La actividad permitió comprender la puesta en servicio de un sistema industrial real y la importancia de revisar el circuito antes de energizarlo.

Ver evidencia

Tablero de comando y protección industrial

Se inspeccionó y completó parcialmente el cableado de un tablero de comando y protección, diferenciando el circuito de mando en baja tensión y el circuito de potencia trifásico.

Esta práctica permitió trabajar con criterios de seguridad eléctrica, relevamiento de conexiones y puesta en marcha controlada del sistema.

Ver evidencia

Circuito de arranque/parada con contactor

En esta actividad se representó y trabajó con un circuito de mando para arranque y parada de motor mediante contactor, pulsadores de START/STOP y contacto de retención.

Esta práctica permitió comprender una lógica básica pero fundamental en sistemas eléctricos industriales.

Ver evidencia

Arranque secuencial y apagado de motores

Se desarrolló un esquema de mando y potencia para el arranque y apagado de motores, incorporando contactores, temporizadores, protecciones, emergencia y señalización.

Esta actividad permitió relacionar la lógica de control con la puesta en servicio de sistemas eléctricos de mayor complejidad.

Ver evidencia

Circuitos neumáticos con cilindros

En las prácticas de neumática se montaron circuitos con cilindros de simple y doble efecto, válvulas, manómetros, reguladores y fuente de aire comprimido.

Estas actividades permitieron interpretar simbología neumática, conectar componentes y verificar el funcionamiento del sistema.

Ver evidencia

Circuito para apertura de silo de pienso

Se diseñó y analizó un circuito neumático/electroneumático para abrir un silo de pienso, utilizando cilindros, válvulas, accionamientos y elementos de control.

La práctica permitió aplicar la lógica de mando neumático a una situación funcional concreta.

Ver evidencia

Accionamientos eléctricos con relés y contactores

Se trabajó con relés, contactores, pulsadores, temporizadores, llave de emergencia y circuitos de mando con retención.

Estas prácticas permitieron comprender la lógica de arranque, parada, enclavamiento y seguridad en sistemas eléctricos de control.

Ver evidencia

¿Cómo desarrollé y cómo aplico esta competencia en mi vida académica?

Desarrollé esta competencia en actividades donde era necesario conectar, revisar y poner en marcha sistemas eléctricos, neumáticos y embebidos. Estas prácticas me ayudaron a valorar el orden en el cableado, la interpretación de esquemas, la revisión previa y la seguridad antes de energizar un sistema.

En mi formación académica, esta competencia me permite enfrentar los trabajos prácticos con mayor responsabilidad, entendiendo que la puesta en servicio no solo implica que el sistema funcione, sino que también debe hacerlo de forma segura, confiable y bien documentada.
